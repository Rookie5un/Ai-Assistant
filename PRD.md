**AI 助手 Web 端系统 PRD（MVP）**

文档版本：`V1.0`  
日期：`2026-03-31`  
适用范围：`Vue + Python 技术栈的 AI 助手 Web 系统首版`

**1. 产品概述**
产品名称：`AI 助手 Web`

产品定位：  
面向个人用户和小团队，提供一个可在线使用的 AI 助手系统，支持多轮对话、知识库问答、文件上传、助手配置和基础管理能力，帮助用户完成信息查询、内容生成、资料总结和工作辅助。

产品愿景：  
让用户在一个统一的 Web 平台内，完成“聊天 + 知识检索 + 助手定制 + 文档问答”这几类高频 AI 使用场景。

**2. 背景与目标**
当前用户使用 AI 时常见问题：

- 对话分散在多个平台，历史记录难管理
- 上传资料后无法稳定做问答
- 不同场景下需要反复写 Prompt
- 缺少适合团队或个人长期使用的统一入口

本期目标：

- 提供稳定的 AI 聊天体验
- 支持历史会话管理
- 支持文件上传和知识库问答
- 支持创建不同用途的“助手角色”
- 提供基础运营和使用统计能力

本期不做：

- 复杂多智能体协作
- 自动长链路任务编排
- 企业级审批流
- 计费与支付系统
- 多租户企业组织架构

**3. 目标用户**
核心用户：

- 个人创作者
- 独立开发者
- 小团队运营人员
- 需要频繁总结文档、写内容、查资料的知识工作者

用户特征：

- 有明确的 AI 使用需求
- 希望保留聊天记录和资料
- 需要比通用聊天产品更贴近自己工作流的系统

**4. 核心使用场景**
场景 1：日常聊天辅助

- 用户输入问题
- 系统调用大模型生成回答
- 支持连续追问和上下文记忆

场景 2：文档问答

- 用户上传 PDF、Word 或文本文件
- 系统自动切片、向量化、入库
- 用户基于文档内容发起提问并获得引用型回答

场景 3：角色化助手

- 用户创建“写作助手”“运营助手”“客服助手”等
- 为助手配置系统提示词、模型、温度、知识库绑定
- 在不同场景中快速切换使用

场景 4：后台管理

- 管理员查看用户数、消息量、Token 使用量、上传文件量和错误日志

**8. 详细功能需求**

**8.1 用户与权限**
功能说明：

- 支持用户注册、登录、退出
- 支持账号密码登录
- 支持 JWT 鉴权
- 支持刷新 Token
- 支持查看和修改个人资料

规则要求：

- 邮箱需唯一
- 密码需加密存储
- 未登录用户不可访问业务页面
- 管理员与普通用户角色分离

验收标准：

- 用户可在 1 分钟内完成注册并进入系统
- 登录状态刷新页面后保持有效
- 非授权接口访问返回明确错误码

**8.2 会话与消息**
功能说明：

- 支持创建、重命名、删除会话
- 支持查看历史消息
- 支持多轮连续对话
- 支持消息流式生成
- 支持中断回答
- 支持重新生成上一条回答

规则要求：

- 每条消息需记录角色、内容、时间、关联会话
- 会话按最近更新时间排序
- 消息发送失败需提示重试

验收标准：

- 用户发送消息后 2 秒内开始流式返回
- 会话切换后能正确恢复上下文
- 删除会话后该会话及消息不可再见

**8.3 助手角色**
功能说明：

- 支持创建助手
- 支持配置名称、描述、系统提示词
- 支持选择模型
- 支持配置温度、最大输出长度
- 支持绑定知识库

规则要求：

- 每个用户可创建多个助手
- 助手配置修改后仅影响后续消息
- 默认提供至少 3 个预设助手

验收标准：

- 用户可在 3 步内完成一个新助手创建
- 切换助手后模型和提示词生效正确

**8.4 文件上传与知识库**
功能说明：

- 支持上传 PDF、DOCX、TXT、Markdown
- 文件上传后显示解析状态
- 文档解析后自动切片、向量化、入库
- 用户可基于知识库提问
- 回答可显示引用来源

规则要求：

- 单文件大小限制可配置
- 仅允许用户访问自己的知识库
- 解析失败需保留失败状态和原因
- 知识库删除后相关切片同步失效

验收标准：

- 用户可成功上传文件并看到处理状态
- 文档处理完成后可用于问答
- 知识库问答结果应与上传资料内容一致
- 回答应尽量附带引用片段或文件名

**8.5 后台管理**
功能说明：

- 查看总用户数、总会话数、总消息数
- 查看模型调用量和 Token 消耗
- 查看上传文件数量和处理结果
- 查看错误日志
- 查看近期活跃用户

规则要求：

- 后台仅管理员可见
- 统计数据支持按日查看
- 关键异常需保留详细日志

验收标准：

- 管理员进入后台后可看到核心概览
- 日统计数据与数据库记录一致
- 异常日志可定位到用户、接口和时间

**9. 核心业务流程**

**9.1 新用户首次使用**
1. 用户注册并登录
2. 进入首页
3. 系统引导创建第一个会话
4. 用户发起首次提问
5. 系统返回流式回答
6. 用户可保存历史记录继续追问

**9.2 知识库问答流程**
1. 用户创建知识库
2. 上传文件
3. 系统解析、切片、向量化
4. 文件状态变为可用
5. 用户在聊天中绑定知识库提问
6. 系统检索相关片段并生成回答
7. 前端展示答案及引用来源

**9.3 助手创建流程**
1. 用户进入助手管理页
2. 点击新建助手
3. 填写名称、描述、提示词
4. 选择模型和参数
5. 绑定知识库
6. 保存后进入聊天页使用

**10. 非功能需求**
性能要求：

- 普通聊天请求平均首字返回时间小于 2 秒
- 页面首屏加载时间小于 3 秒
- 支持至少 100 个并发在线用户的初期运行

安全要求：

- 密码加密存储
- 文件上传类型校验
- 接口鉴权
- 请求频率限制
- 敏感日志脱敏

可用性要求：

- 系统可用率目标不低于 99%
- 核心异常可追踪
- 文件处理失败可重试

兼容性要求：

- 支持 Chrome、Edge、Safari 主流浏览器
- 支持桌面端优先，兼容移动端浏览

**13. 风险与应对**
风险点：

- 模型响应速度不稳定
- 文档解析效果不佳
- RAG 命中率不足
- 流式输出体验不稳定
- 上线后成本超预期

应对方案：

- 增加超时和重试机制
- 文件处理做异步队列
- 优化切片策略和召回策略
- 首版先控制模型种类
- 增加 Token 和文件使用量监控

**14. MVP 验收标准**
满足以下条件可判定首版可上线：

- 用户可注册登录并稳定使用
- 支持新建会话和多轮聊天
- 支持流式回复
- 支持创建助手并保存配置
- 支持上传文件并进行知识库问答
- 管理员可查看基础运营数据
- 核心功能在主流浏览器可正常运行
- 系统具备基本日志、鉴权和异常处理能力
